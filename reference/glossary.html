Glossary

Context & Harness Engineering  ·  Updated 2026-06-08

Core Equation
Agent
Model + Harness. The full system, not just the LLM.
Model
The neural network (e.g. Claude Sonnet 4.6). Fixed — you don't change it.
Harness
Everything wrapped around the model: prompts, tools, hooks, skills, memory, MCP servers. This is what you engineer.
Six Harness Layers
CLAUDE.md
Project-level rules injected into every session. Write operation. Lives in repo root.Layer 1 — System knowledge
Tool surface
What actions Claude can take: Bash, Read, Edit, Write, Agent…Layer 2 — Actions
MCP server
Model Context Protocol. External systems wired in as tools (GitHub, Jira, APIs). Extends the tool surface.Layer 3 — External systems
Skill
A reusable, triggerable workflow — a Markdown file with instructions Claude reads when invoked. Invoked via /skill-name.Layer 4 — Reusable workflows
Hook
Shell command that fires at a lifecycle event (PreToolUse, PostToolUse, PreCommit…). The harness's enforcement layer.Layer 5 — Lifecycle enforcement
Memory
Files persisted between sessions. Written by Claude or you; injected into future sessions via ~/.claude/projects/.../memory/.Layer 6 — Cross-session state
Four Context Strategies
Write
Persist information outside the context window so it can be retrieved later. CLAUDE.md, memory files, task files.
Select
Choose which information to inject at inference time. Skills loaded only when triggered. RAG retrieval.
Compress
Reduce token count without losing meaning. Context compaction. Summaries in memory instead of raw transcripts.
Isolate
Run sub-tasks in separate agent contexts so they don't contaminate each other. Subagent spawning via Agent().
Context Engineering Concepts
Context window
The finite token buffer the model sees at inference time. Everything in here shapes the output — nothing outside it does.
Context engineering
Deliberately designing what goes into the context window: what to include, exclude, compress, and when to retrieve it.
Prompt engineering
Crafting the phrasing of a query. A subset of context engineering — upstream context often matters more than phrasing.
RAG
Retrieval-Augmented Generation. Fetch relevant documents from external storage and inject them as context at query time.
Ratchet principle
Every agent mistake → permanent harness rule. Treat failures as config signals, not model limitations. The harness only gets tighter.
Tool-Agnostic Harness
AGENTS.md
Vendor-neutral, cross-tool project instructions file. Place at repo root. Read by Claude Code, Cursor, GitHub Copilot, Windsurf, Aider, and more. The portable equivalent of CLAUDE.md. Started as an OpenAI initiative (Aug 2025); now the de-facto cross-tool standard. Lesson 12
.cursor/rules/
Cursor's project rules directory. Each .mdc file has YAML frontmatter with globs, alwaysApply, and description. Glob-triggered rules only load for matching file types. Legacy .cursorrules at root still works. Lesson 12
copilot-instructions.md
GitHub Copilot's repo-wide instruction file at .github/copilot-instructions.md. Path-specific variants live at .github/instructions/*.instructions.md. Propagates from organization → repository → personal. Lesson 12
.windsurfrules
Windsurf (Cascade) project rules file. Plain text/Markdown at project root. Team-shared, version-controlled. Separate from Cascade Memories (which are automatic and local). Lesson 12
Cascade Memories
Windsurf's automatic cross-session memory. Cascade builds context as you work. Unlike Claude Code memory (manual write), Cascade Memories accumulate implicitly. Workspace-scoped, not version-controlled. Lesson 12
MCP config key variance
Critical gotcha: Claude Code and Cursor use "mcpServers" as the root key. VS Code / GitHub Copilot uses "servers" (in .vscode/mcp.json). Copilot MCP only works in Agent mode. Lesson 12
Loop Engineering
Loop engineering
Designing the system that prompts an agent on a schedule, instead of prompting it yourself. The leverage shifts from crafting individual prompts to designing the control system. Popularized by Addy Osmani, mid-2026. Lesson 13
Ralph Loop
The original pattern: while true; do claude -p "prompt"; done with state persisted to disk between iterations. Documented by Geoffrey Huntley, July 2025. Lesson 9, 13
Ralph Wiggum failure mode
A loop's agent emits its "done" signal early, on a half-finished job, and the loop exits declaring success on wrong work. Fixed with a separate checker sub-agent — never let the maker grade its own homework. Lesson 13
Maker/checker split
One agent (or context) does the work; a separate agent (or context) verifies it independently. The core defense against the Ralph Wiggum failure mode. Lesson 13
Six loop components
Automations (scheduling), Worktrees (isolation), Skills (project knowledge), Connectors (MCP), Sub-agents (maker/checker), State/Memory (persistence). Five of six map directly to harness layers from Lessons 1–12 — only Automations is new. Lesson 13
L1 / L2 / L3 rollout
Phased loop deployment: L1 = report only, zero changes. L2 = proposes changes via PR, human approves. L3 = unattended within a tight allowlist. Every new loop starts at L1, regardless of how low-risk the task seems. Lesson 13
Comprehension debt
Code shipped by a loop that no one on the team has read. Accumulates silently; becomes a liability the first time it needs to change. Lesson 13
Cognitive surrender
Accepting loop output without judgment because it's usually right. The loop amplifies whatever judgment (or lack of it) you bring to reviewing its output. Lesson 13
Token Compression Tools
RTK
Rust Token Killer. CLI proxy that intercepts shell command output and compresses it before Claude sees it. Install with brew install rtk, wire with rtk init -g. Saves 60–92% on git, test runners, ls, find, and 100+ other commands. Lesson 11
Graphify
Builds an AST-level knowledge graph of your codebase. Claude queries the graph instead of reading files. A BFS subgraph query returns ~2k tokens vs. ~50k–100k tokens for naive file reading on large repos. Run once with npx graphify index. Lesson 11
LLMLingua
Microsoft Research prompt compression. A small model scores each token by perplexity; low-score tokens are surgically removed. 4–20× compression on RAG-retrieved documents. Not summarization — the remaining text is the original tokens. Lesson 11
Repomix
Packs a repository into a single LLM-optimized Markdown or XML file. Strips binaries, node_modules, build artifacts. Used with the Isolate strategy: pack → subagent → summary. Lesson 11
Caveman mode
Built-in Claude Code skill (/caveman) that compresses Claude's output tokens ~75% by stripping grammatical overhead. Full technical accuracy preserved. Toggle per-session. Lesson 11
LSP
Language Server Protocol. Compiler-backed symbol navigation: go-to-definition, find-references, hover types. Replaces grep-and-read loops with single precise queries. 80–95% token savings on navigation tasks. Lesson 11
Claude Code Specifics
settings.json
The harness config file. Controls permissions, hooks, environment vars. Lives at ~/.claude/settings.json (global) or .claude/settings.json (project).
PreToolUse hook
Runs before any tool call. Can block the action by returning non-zero exit code. Used for safety gates.
PostToolUse hook
Runs after a tool call completes. Used for side effects: logging, linting, running tests.
Subagent
A child agent spawned via Agent(). Gets its own fresh context. The primary isolation primitive.
Ralph Loop
Multi-session continuation pattern. Agent writes state to disk at end of session; resumes from that state in the next. Enables long-horizon work.
