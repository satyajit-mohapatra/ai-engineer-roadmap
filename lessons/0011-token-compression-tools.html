Token Compression Tools

Six tools that clean, compress, and repackage context before it ever reaches the model — saving 60–90% on token costs.

The Problem

In Lesson 7 you learned that tool results are the biggest context consumer in most sessions. A git status that returns 200 lines dumps 1,500+ tokens into the window. A test run with 80 failures returns 10,000 tokens of stack traces. A recursive directory listing: 3,000 tokens of paths most of which are irrelevant.

The Compress strategy from Lesson 8 says: reduce token count without losing meaning, before information lands in the window. These six tools operationalize that strategy — each attacking a different source of token bloat.

Key Framing
These tools don't make the model smarter. They make the input smaller without making it less useful. A 30-minute Claude Code session typically uses 100k–150k tokens. With the right compression stack, the same session can run on 15k–30k tokens — same quality, 80% lower cost.

The Tools

RTK
RTK — Rust Token Killer
CLI proxy that intercepts shell output and compresses it before Claude sees it
60–92%
token reduction

RTK sits between your shell and Claude Code. When Claude runs a Bash command, RTK intercepts the output, applies four compression strategies (smart filtering, grouping, truncation, deduplication), and hands a compressed version to Claude. Claude never sees the raw output.

Install & Setup brew install rtk rtk init -g # auto-installs hooks into ~/.claude/settings.json # Restart Claude Code — hooks rewrite Bash commands transparently

What it compresses: git status, git log, git diff, ls, find, cat, grep, test runners (pytest, cargo test, go test), linters (ESLint, TypeScript), container logs, AWS CLI. Over 100 supported commands.

How it works at each command:

Before RTK — git status
On branch feature/auth Your branch is ahead of 'origin/feature/auth' by 3 commits. Changes not staged for commit: (use "git add <file>..." to update) (use "git restore <file>..." to discard) modified: src/auth/login.js modified: src/auth/session.js modified: tests/auth.test.js Untracked files: (use "git add <file>..." to include) src/auth/refresh.js no changes added to commit
~180 tokens
After RTK — same command
branch: feature/auth (+3 ahead) modified: src/auth/login.js src/auth/session.js tests/auth.test.js untracked: src/auth/refresh.js
~35 tokens (81% saved)
Intercepts shell output Rewrites Bash commands via hook Track savings with rtk gain

Limitation: Claude Code's built-in Read/Grep tools bypass the hook — they don't auto-compress. For those, use Bash equivalents or call rtk cat / rtk grep explicitly.

G
Graphify
Indexes your codebase as a queryable knowledge graph — no more scanning raw files
99%
on large codebases

Graphify runs a one-time AST analysis of your codebase and produces two artifacts: GRAPH_REPORT.md (a structured map of functions, classes, dependencies, and call sites) and graph.json (the queryable index). Claude reads the map first, then asks targeted questions against the graph instead of scanning files.

Install & Index npx graphify index # run once in project root # produces GRAPH_REPORT.md + graph.json npx graphify query "find all callers of validateSession"

The compression math: On a 500k-token codebase (roughly 400k LOC), reading relevant files naively might load 50k–100k tokens per task. A Graphify BFS subgraph query for the same answer: ~2,000 tokens. The graph is built once and reused across sessions — it persists on disk.

Add to CLAUDE.md:

## Codebase Navigation
A Graphify knowledge graph is at GRAPH_REPORT.md and graph.json.
Before reading source files to understand architecture, query the graph first:
  npx graphify query "<your question>"
Only open individual files when the graph query points you to a specific location.
AST-level code understanding Persists across sessions Re-run after major refactors
LL
LLMLingua
Surgically removes low-value tokens from prompts using a smaller model's perplexity scores
4–20×
prompt compression

LLMLingua (Microsoft Research) runs a small, cheap model over your prompt and identifies tokens that, if removed, wouldn't meaningfully change the large model's output. It's not summarization — it's surgical removal of low-perplexity tokens. The compressed prompt is semantically equivalent but much shorter.

Install pip install llmlingua 
Python — basic usagefrom llmlingua import PromptCompressor

compressor = PromptCompressor(
    model_name="microsoft/llmlingua-2-bert-base-multilingual-cased-meetingbank",
    use_llmlingua2=True
)

result = compressor.compress_prompt(
    context=long_context,       # your RAG docs, system prompt, etc.
    instruction=question,       # the actual question
    target_token=200,           # target compressed length
    rank_method="longllmlingua"
)

compressed = result["compressed_prompt"]
# Pass `compressed` to Claude instead of the raw context

Best for: RAG pipelines where retrieved documents are long. Compressing retrieved context before injecting it can cut RAG overhead by 60–94% with minimal quality loss. Not recommended for short prompts or code — perplexity-based removal can strip important syntax.

Runs on CPU (small model) Works on any text LLMLingua-2 is 3–6× faster
RM
Repomix
Packs an entire repository into one LLM-optimized file — with binary exclusion, ignore patterns, and structure maps
~40%
vs. naive concatenation

Repomix converts a repo into a single Markdown file structured for LLM consumption: directory tree first, then each file's content with clear separators. It strips binaries, node_modules, build artifacts, and respects .gitignore. The result can be injected into a subagent's context for whole-codebase analysis tasks.

Install & Pack npx repomix # pack current repo → repomix-output.xml npx repomix --include "src/**" # only src/ npx repomix --style markdown # output as markdown npx repomix --output-show-line-numbers # add line numbers for precise references

Pattern: Pack the repo → inject into a subagent for a whole-codebase task → get a structured summary back. The subagent's context holds the entire codebase; your main session context holds only the result.

Pairing with Isolate strategy# Pack relevant subdirectory
npx repomix --include "src/auth/**" --style markdown > /tmp/auth-pack.md

# Subagent brief includes the pack
Brief: "The auth module code is in /tmp/auth-pack.md.
Read it, then identify all places where session tokens
are stored without expiry. Return a list of file:line references."
Strips build artifacts Respects .gitignore Works with remote repos via URL
CV
Caveman Mode
Ultra-compressed communication — strips filler, articles, and pleasantries from Claude's output
~75%
output token reduction

Caveman compresses Claude's responses, not the context. It strips grammatical completeness and stylistic prose while preserving full technical accuracy. Useful when you're doing rapid iteration and don't need full sentences — you just need the facts.

Invoke (it's a built-in Claude Code skill) /caveman # activate for this session /caveman off # return to normal mode
Normal output
I've reviewed the authentication module and found three issues that need attention. First, the session token is stored without an expiry time, which creates a security risk...
~60 tokens
Caveman output
auth issues: - token: no expiry → security risk - refresh: no rate limit - logout: doesn't invalidate server-side
~20 tokens (67% saved)

Best for: Exploratory sessions, debugging loops, rapid Q&A where you want facts not prose. Switch off for final output — reports, documentation, PR descriptions — where full sentences matter.

Reduces output tokens only Full technical accuracy preserved Toggle per-session
LSP
LSP — Language Server Protocol
Compiler-backed symbol navigation instead of grep-and-read loops
80–95%
vs. grep loops

When Claude wants to find where a function is defined or all its callers, the naive approach is: grep across the project, read matching files, scan for context. Each step dumps tokens. LSP replaces this with a single precise query: go-to-definition, find-references, hover type — one call, one result, no scanning.

Claude Code has LSP integration built in via the mcp__lsp MCP server (when configured). Alternatively, expose LSP queries through a shell wrapper Claude can call directly.

Configure (example — TypeScript) npm install -g typescript-language-server typescript # Then wire as MCP server or expose via Bash wrapper: npx typescript-language-server --stdio
Without LSP — grep loop
grep -r "validateSession" src/ → 12 files matched → Read auth/middleware.js (800 tokens) → Read auth/session.js (600 tokens) → Read tests/auth.test.js (1200 tokens) ...
~5,000 tokens to find one def
With LSP — one query
lsp definition validateSession → src/auth/session.js:47 lsp references validateSession → middleware.js:23, router.js:89, tests/auth.test.js:14 (3 refs)
~40 tokens. Done.
Compiler-accurate (not text search) Works across renames & imports Built in to most language servers

The Compound Effect
RTK alone on a 30-minute session: 118k → 24k tokens (80% saved). Add Graphify for codebase navigation and LSP for symbol lookups, and the same session might run on 8k–12k tokens. These savings aren't additive — they're multiplicative, because each tool prevents a category of bloat that would otherwise compound across turns.

What to Hold Onto

  1. RTK — install it now and restart Claude Code. Immediate 60–90% savings on every CLI command, zero configuration needed beyond rtk init -g.
  2. Graphify — the highest-leverage tool for large codebases. One index run; every future session benefits. Add the query instruction to CLAUDE.md.
  3. LSP — replace grep loops with compiler-accurate symbol queries. 80–95% savings on navigation tasks.
  4. LLMLingua — purpose-built for RAG token bloat. Not useful outside retrieval pipelines, but essential inside them.
  5. Repomix — the right pairing with the Isolate strategy. Pack → subagent → summary. Never put a full repo in the main session.
  6. Caveman — a built-in skill. Use during fast iteration loops; turn off for any deliverable output.

The fastest ROI: install RTK first (brew install rtk && rtk init -g), then run Graphify on your project (npx graphify index) and add the query instruction to your CLAUDE.md. These two changes alone will cut most sessions by 70–80%.
