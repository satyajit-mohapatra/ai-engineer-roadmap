Multimodal AI

Models that see, hear, and create — not just read and write. The same API shape you know, now with images and audio flowing through it.

The mental shift
"Multimodal" = a model that handles more than text — images, audio, sometimes video — as input, output, or both. Under the hood, other modalities get converted into tokens the same transformer processes (Lesson 15). For you, the API is mostly unchanged: you just attach an image or audio file.

Image understanding (the most useful for you)

A vision-capable model can read a chart, a screenshot, a scanned PDF, or a handwritten note. The call is the familiar messages format — you just add an image part:

python — analyze a chart screenshotresp = client.chat.completions.create(model="gpt-5", messages=[{
  "role":"user", "content":[
    {"type":"text", "text":"What pattern is forming on this chart?"},
    {"type":"image_url",
     "image_url":{"url":"data:image/png;base64,..."}}]}])
A real tie to your work
Vision is genuinely useful in trading workflows: feed a chart screenshot and ask the model to describe the setup, extract numbers from a broker statement (OCR), or read a table from a PDF report into structured data (combine with Structured Outputs, Lesson 18). It's not a trading oracle — but it's a strong "read this image into text/JSON" tool.

Speech & generation

  • Whisper / STT — transcribe audio to text. One API call: send the file, get the transcript. Good for meeting notes, voice commands, podcast indexing.
  • TTS — turn text into natural speech. Pick a voice, send text, get audio. Powers voice assistants and accessibility.
  • Image generation — DALL·E and friends turn a text prompt into an image. Quality is prompt-sensitive; iterate like you would any prompt.

Orchestrating multimodal apps

For apps that chain modalities (e.g. transcribe → summarize → generate an image), frameworks like LangChain and LlamaIndex have multimodal modules so you don't hand-wire every step. But each individual call is still the simple "attach the media" pattern above — don't let frameworks obscure that.

Quiz · Question 1 of 2
You want to extract the strike prices and expiry dates from a scanned PDF options statement into clean JSON. Which combination of capabilities does this need?
A
Text-to-speech plus image generation, to read the file aloud and redraw it
B
Image understanding (vision/OCR) plus structured output to enforce the JSON shape
C
Speech-to-text plus a reranker, since PDFs are processed as audio streams
D
Video understanding, because a multi-page PDF counts as a sequence of frames
Correct. A scanned PDF is an image, so you need image understanding / OCR to read the values, then Structured Outputs (Lesson 18) to force them into a strict JSON schema. This image→text→structured combination is one of the most practical multimodal patterns. The other options mix in unrelated modalities (audio, video, generation).
A scanned PDF is an image: use vision/OCR to read it, then Structured Outputs to enforce JSON (option B). TTS, STT, video, and image generation don't apply to reading values out of a scanned document.
Quiz · Question 2 of 2
How does sending an image to a vision model differ, at the API level, from a normal text chat call?
A
Completely different — vision requires a separate protocol and its own SDK
B
Largely the same — you use the messages format and add an image content part
C
You must first convert the image to text yourself before the model can read it
D
Images can only be sent to fine-tuned models, never to standard hosted ones
Correct. It's the same Chat Completions / messages shape you learned in Lesson 18 — you just include an image content part alongside the text. The model handles converting the image into tokens internally. That continuity is why mastering the base API pays off across every modality.
It's the same messages format with an added image part (option B) — no separate protocol, no manual image-to-text step, and no fine-tuning requirement. The model tokenizes the image internally.
