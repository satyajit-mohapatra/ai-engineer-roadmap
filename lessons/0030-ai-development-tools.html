You already know how to evaluate these
Lesson 20 (tool-agnostic harness engineering) taught you that CLAUDE.md, .cursor/rules, Copilot custom instructions, Windsurf memories, and AGENTS.md are the same idea in different syntax. Switching tools is a config-translation problem, not a relearning problem. That's a Phase 1 superpower applied to this whole table.

What separates them (the axes that matter)

  • Autonomy — inline completion (Copilot) → chat → agent that edits many files and runs commands (Claude Code, Codex). More autonomy = more leverage and more need for the gating you learned in Lesson 21.
  • Harness depth — hooks, skills, custom tools, memory, subagents. This is where Claude Code is deep and where your Phase 1 skills directly pay off.
  • Context handling — how the tool selects, compresses, and isolates context (Lessons 15–17) determines reliability on big codebases.
  • Model flexibility — can you bring your own model / route providers (Lesson 28)?
