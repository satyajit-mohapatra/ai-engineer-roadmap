Case Study: Daily Report Pipeline

Let's design a complete harness for a common workflow: a daily report that fetches data from multiple sources, filters by criteria, and produces a formatted output file.

Step 1 — Define the workflow

What needs to happen, in order:

  1. Fetch data from N sources in parallel (each source is independent)
  2. Filter by project-specific criteria
  3. Score and rank results
  4. Format as a structured report
  5. Save to output/daily-report-YYYY-MM-DD.md

Step 2 — Assign harness layers

HARNESS DESIGN# Layer 1 — CLAUDE.md
Add to project CLAUDE.md:
  - Report config lives in src/config/reportConfig.js
  - Output directory: output/  (always writable)
  - Date format for filenames: YYYY-MM-DD

# Layer 4 — Skill: /daily-report
Trigger: user types /daily-report
Body:
  1. Read source list from data/sources.txt
  2. Spawn parallel subagents (one per source) — each fetches
     data and returns top results as JSON
  3. Collect results, filter by criteria, rank by score desc
  4. Write formatted report to output/daily-report-{date}.md
  5. Print summary table to console

# Layer 5 — Hooks
PostToolUse on Write tool:
  If file matches output/daily-report-*.md → run validation script
  Validates: file exists, has content, no empty required fields
  async: true — log result but don't block

# Layer 6 — Memory
Feedback memory: "User prefers reports sorted by score descending,
not alphabetically. Confirmed when reviewing first output."
Reference memory: "Source list is at data/sources.txt"

Step 3 — Design the subagent brief template

SUBAGENT BRIEF TEMPLATE (one per source)You are a read-only data fetching agent.

**Goal**: Fetch data for {SOURCE} and return the top 3 results
matching our criteria.

**Context**:
- Project: /path/to/your-project
- Fetch using: the fetch script at scripts/fetch.sh {SOURCE}
- Filter criteria: score ≥ threshold defined in src/config/reportConfig.js

**Constraints**:
- Run only the fetch script — no other commands
- Do not write any files

**Output** (return exactly this JSON, no prose):
{
  "source": "{SOURCE}",
  "results": [
    {"id": "...", "score": N, "label": "...", "fetched_at": "ISO timestamp"}
  ]
}

Step 4 — Apply the four strategies

DecisionStrategyImplementation
Filter criteria persist across sessionsWriteAdd to CLAUDE.md or a config file
N sources scanned independentlyIsolateN parallel subagents, one per source
Each subagent returns JSON not transcriptCompressExplicit output format in brief

The Diagnosis Framework

When a harness isn't working, use this sequence:

SymptomLayer to checkStrategy
Claude forgets rules between sessionsLayer 1 or 6 — CLAUDE.md or memoryWrite missing
Context fills up, quality degrades mid-sessionTool results, conversation lengthCompress or Isolate missing
Claude uses wrong tools or ignores available onesLayer 3 or 4 — MCP or skillsSelect guidance missing in CLAUDE.md
Claude does something dangerous (force-push, secret leak)Layer 5 — hooksPreToolUse block hook missing
Subagent returns irrelevant or verbose resultBrief qualityOutput format not specified
Same mistake repeated across sessionsLayer 6 — memoryFeedback memory not written
The Ratchet in Full
Every time you diagnose a failure and fix a harness layer, the agent gets permanently better. CLAUDE.md gets one cleaner rule. A hook blocks one more edge case. A feedback memory prevents one repeat mistake. A subagent brief gets one more constraint. This compounds — a harness you've been maintaining for 3 months is dramatically more reliable than a fresh one. The ratchet only tightens.

Synthesis Exercise: Design Your Own

Your Turn — Design a Harness

Pick one workflow from your own project that Claude currently handles poorly or inconsistently. Work through each step.

1
Name the workflow
What does it do? What's the trigger? What's the expected output? (e.g. "Run a data pipeline, validate the output, and produce a formatted report")
2
List what goes wrong today
Where does Claude fail, forget, or produce inconsistent output? List 3–5 concrete failures.
3
Map each failure to a layer + strategy
For each failure: which harness layer is missing or wrong? Which strategy is absent? Use the diagnosis table above.
4
Design the fix for each layer
Write the CLAUDE.md addition, the skill body, the hook script, or the memory file that addresses each failure. Be specific — no "add a rule" without the actual rule text.
5
Implement one layer this session
Pick the highest-leverage fix and implement it now. Ask Claude to help. The ratchet starts with the first commit.
